# Topological Cartographer (`ca/`)

The Topological Cartographer is an interactive Python web application designed to analyze large-scale survey data generated by the S34/S35 Genetic Algorithm (GA). By offloading heavy data processing from the browser to a local Python environment, this tool transforms the GA from a "blind" optimization engine into a powerful, guided discovery engine.

It utilizes Principal Component Analysis (PCA) to create a topographical map of the fitness landscape, allowing users to visualize the solution space, identify distinct families of high-performing LEGO sorter designs, and understand the correlations between genetic parameters and physical performance.

## ✨ Features

* **Interactive PCA Dashboard:** Dynamic exploration of the fitness landscape through 2D and 3D scatter plots.
* **Large File Handling (Reservoir Sampling):** Safely loads and processes a random sample (default 75,000 records) from massive, multi-gigabyte `.jsonl` files without crashing or exceeding system memory.
* **Automated Data Flattening:** Automatically unpacks and flattens nested JSON chromosome structures (e.g., `funnel_profile`, `complexRamps`, and `pegMatrices`) into a 1D format suitable for scikit-learn's machine learning analysis.
* **Dynamic Filtering:** Includes an interactive range slider to filter the visualized landscape by specific fitness score ranges, isolating the most promising design regions.
* **Statistical Insights:** Generates an explained variance bar chart and a PCA component loadings heatmap to reveal which physical design choices are driving system performance.

## 📦 Requirements

The application relies on several Python libraries for data manipulation, machine learning, and dashboard rendering. 

Install the dependencies using pip:
```bash
pip install pandas scikit-learn plotly dash pyarrow
```

## 🚀 Usage Workflow

1. **Generate Data:** Run the S35 Genetic Algorithm in the browser and use the UI to download the surveyor data as a `.jsonl` file.
2. **Launch the Cartographer:** Open your terminal, navigate to the `ca` directory, and execute the script:
   ```bash
   python cartographer.py
   ```
3. **Access the Dashboard:** Open your web browser and navigate to the local server address, typically `http://127.0.0.1:8050/`.
4. **Upload and Explore:** Drag and drop your `.jsonl` survey log into the upload box on the dashboard. 
5. **Analyze:** * Adjust the "Sample Size" if needed before uploading.
   * Use the dropdown controls to select which of the 10 calculated principal components to plot on the X, Y, and Z axes. 
   * Switch between 2D and 3D view modes.

## 🏗️ Architecture Context

This tool represents Stage 2 of the "Decoupled Analysis Pipeline" for the S35 architecture.
* **Stage 1 (Browser - `ga/`):** The GA worker acts as the "Surveyor," running parallel physics simulations and bundling the results into JSONL chunks to keep browser memory usage low.
* **Stage 2 (Python - `ca/`):** This script acts as the "Cartographer," aggregating those chunks, performing computationally heavy PCA, and rendering the interactive topographical map.
